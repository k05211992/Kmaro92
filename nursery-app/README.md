# 🌿 Nursery App — Каталог растений и генератор КП

MVP веб-приложения для питомника растений: подбор растений по параметрам, формирование коммерческих предложений и выгрузка в Excel.

---

## Быстрый старт

### 1. Установка зависимостей

```bash
cd nursery-app
npm install
```

### 2. Генерация примера Excel-каталога

```bash
npm run generate-sample
```

Команда создаст файл `public/sample_catalog.xlsx` с примером из 14 растений (деревья, кустарники, многолетники, розы).

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000)

### 4. Production-сборка

```bash
npm run build
npm start
```

## Основной сценарий

### Менеджер

1. Переключите режим в **«Менеджер»** (правый верхний угол).
2. На главной странице нажмите **«Загрузить каталог Excel»**.
3. Загрузите файл `.xlsx` с растениями.
4. Перейдите в **Каталог**.
5. Фильтруйте растения, добавляйте в КП кнопкой **«В КП»**.
6. Откройте панель КП (кнопка в хедере).
7. Укажите количество, скидку, тип цены (розница/опт) для каждой позиции.
8. Выберите тип доставки.
9. При желании укажите данные клиента.
10. Нажмите **«Сформировать КП (Excel)»** — файл скачается автоматически.

### Клиент / Ландшафтный дизайнер

1. Работайте в режиме **«Клиент»** (по умолчанию).
2. Каталог уже загружен менеджером.
3. Фильтруйте и добавляйте позиции в подборку.
4. Сформируйте предварительный список (Excel КП).

## Формат Excel-каталога

Первая строка — заголовки колонок (регистр не важен):

| Колонка | Тип | Обязательная | Описание |
|---|---|---|---|
| `id` | строка | нет | Уникальный ID (генерируется автоматически если пусто) |
| `category` | строка | **да** | Категория (Деревья, Кустарники, Многолетники...) |
| `subcategory` | строка | нет | Подкатегория |
| `name` | строка | **да** | Название растения |
| `latin_name` | строка | нет | Латинское название |
| `variety` | строка | нет | Сорт / вид |
| `size` | строка | нет | Размер (Н 1–1.5 м) |
| `container` | строка | нет | Контейнер (С3, С5, С10...) |
| `price_retail` | число | **да** | Розничная цена, руб. |
| `price_wholesale` | число | нет | Оптовая цена, руб. |
| `discount_default` | число | нет | Скидка по умолчанию, % (0–100) |
| `stock_qty` | число | нет | Количество в наличии, шт. |
| `stock_status` | строка | нет | `in_stock` / `low` / `order` / `out` |
| `light` | строка | нет | `sun` / `partial_shade` / `shade` |
| `moisture` | строка | нет | `dry` / `moderate` / `wet` |
| `height_group` | строка | нет | `under_20` / `20_50` / `50_100` / `over_100` |
| `frost_zone` | строка | нет | Зона морозостойкости USDA (2, 3, 4–5...) |
| `lifespan_group` | строка | нет | Срок жизни на одном месте |
| `flower_color` | строка | нет | Цвет цветков |
| `leaf_color` | строка | нет | Окраска листвы |
| `natural_garden` | boolean | нет | Подходит для природного сада (true/false) |
| `medicinal` | boolean | нет | Лекарственное растение (true/false) |
| `description` | строка | нет | Описание |
| `notes` | строка | нет | Примечания / рекомендации |
| `image_url` | строка | нет | URL фото |

### Допустимые значения полей

**stock_status**: `in_stock`, `low`, `order`, `out` (или по-русски: «в наличии», «мало», «под заказ», «нет»)

**light**: `sun`, `partial_shade`, `shade` (или: «солнце», «полутень», «тень»)

**moisture**: `dry`, `moderate`, `wet` (или: «сухо», «умеренно», «влажно»)

**height_group**: `under_20`, `20_50`, `50_100`, `over_100` (или: «до 20», «20-50», «50-100», «100+»)

## Технические решения (MVP Assumptions)

| Решение | Обоснование |
|---|---|
| Данные хранятся в памяти | MVP без БД. При перезагрузке страницы — загрузить Excel снова |
| Авторизация — режимы без логина | MVP. Переключатель клиент/менеджер доступен всем |
| Excel как источник данных | Менеджер обновляет каталог загрузкой файла |
| Доставка — фиксированная или вручную | MVP. Зонирование и расчёт по км — следующий этап |
| Выгрузка КП — без стилей | SheetJS Community Edition без форматирования ячеек |
